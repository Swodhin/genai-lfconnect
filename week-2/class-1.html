Concept 01

It's Just... Next-Token Prediction

Every output an LLM produces — an essay, a poem, a line of code — is built one small piece at a time, by repeatedly asking one question.

  • At each step, the model looks at all the text so far and asks: "what token is most likely to come next?"
  • It doesn't plan the whole answer up front — the next piece is chosen, appended, and the question is asked again.
  • This is the same core skill as your phone's autocomplete — just trained on far more text, with a far larger model.
  • There is no separate "understanding module" — fluent-looking language emerges from this one repeated act.
# the model's job, one step
prompt = "The capital of France is"
next_token = model.predict(prompt)
# next_token → " Paris"

prompt += next_token
# ...repeat until the model emits a stop token
Concept 02

From One Word to a Sentence

This "predict, append, repeat" cycle is called autoregressive generation — each new token is generated conditioned on every token that came before it.

Thequickfoxjumpsover?
  • Faint curved lines = the model weighing how much each earlier word matters for predicting the next one.
  • Generation stops when the model predicts a special end-of-sequence token, or a length limit is hit.
Concept 03

Training, at a High Level

The prediction skill has to come from somewhere. It's learned by adjusting billions of internal numbers ("parameters") so predictions get closer to real text.

  • Self-supervised: training text already contains the "answer" — the model hides the next word and grades itself for free, no human labeling needed.
  • Massive scale: trillions of words from books, websites, code, and articles — far more than any person reads in a lifetime.
  • Gradual adjustment: every wrong guess nudges the model's internal weights slightly, repeated billions of times, until predictions become fluent.
  • Frozen snapshot: once training ends, the model has a knowledge "cutoff" date — it doesn't know about events after that point.
Concept 03, continued

Three Stages Turn Prediction into a Helpful Assistant

Raw next-token prediction alone produces a model that completes text plausibly — not one that follows instructions politely. Two more stages fix that.

STAGE 1

Pretraining

Learn general language patterns, facts, and reasoning shapes by predicting text across the open web.

STAGE 2

Supervised Fine-Tuning

Show the model curated example conversations of good instruction-following, so it learns the assistant "shape."

STAGE 3

Human Feedback (RLHF)

Humans rank multiple model responses; the model is tuned to prefer answers people rate as more helpful and safe.

Concept 04

Architecture Intuition: the Transformer

Modern LLMs are built from transformer blocks, whose signature ingredient is a mechanism called self-attention.

  • Self-attention lets every word in a sentence directly "look at" every other word to decide what's relevant.
  • Older architectures (RNNs) read text strictly left to right, one word at a time — slow, and prone to forgetting distant words.
  • Transformers process a whole sequence in parallel, which is dramatically faster to train on modern hardware.
  • Stacking dozens of attention layers lets the model build up increasingly abstract representations of meaning.

Concept 04, continued

What Attention Buys You: Disambiguation

Attention is how the model figures out which meaning of an ambiguous word applies, using distant context clues.

"I deposited cash at the bank."

Attention weighs "deposited" and "cash" heavily — the model resolves bank as a financial institution.

"We sat by the river bank."

Here "river" pulls the attention weight instead — the same word bank now means the edge of a river.

  • This is also how the model tracks long-range dependencies — a pronoun on line 40 can still "attend to" the name introduced on line 1.

Toy Example: Predicting the Next Word

The companion notebook builds a miniature, from-scratch version of next-token prediction using nothing but word-frequency counts — no API key required.

# tiny toy model: count what follows each word
corpus = "the cat sat the cat ran the dog sat"
from collections import defaultdict, Counter
follows = defaultdict(Counter)
words = corpus.split()
for a, b in zip(words, words[1:]):
  follows[a][b] += 1
# follows["the"] → Counter({'cat': 2, 'dog': 1})
  • Given the word "the", this toy counts how often each word follows it in the training text.
  • "Cat" appears twice after "the," "dog" once — so the toy model predicts "cat" as most likely.
  • Real LLMs replace word counts with billions of learned parameters, but the underlying question is identical.
  • Open the notebook to run this yourself and try feeding it a different tiny corpus.
Common Pitfalls

What an LLM Is Not

The fluent, confident tone of LLM output invites a few tempting but incorrect mental models. Watch for these.

"It understands like a human"

It manipulates statistical patterns in language — it has no beliefs, senses, or goals of its own.

"It's just a fancy search engine"

It doesn't look anything up at inference time by default — it generates text from learned patterns, which can be wrong.

"Bigger always means smarter"

Data quality, fine-tuning, and task fit often matter as much as raw parameter count.

"It remembers our last conversation"

Without an external memory system, each conversation starts fresh — apparent memory comes from what's re-sent as context.

Key Takeaways

  • 02Training happens in stages: self-supervised pretraining, then fine-tuning, then human-feedback alignment.
