Concept

A Prompt Is a Specification, Not a Question

A prompt is the entire instruction set a model receives before it starts generating — there is no other channel for intent.

  • The model only knows what's in the prompt — it has no memory of what you meant to say.
  • Ambiguity in the instructions becomes ambiguity — or invention — in the output.
  • Small wording changes, like a missing word or an unclear pronoun, can shift the entire response.
  • Good prompting is drafting: write, read the output critically, revise, repeat.
  • Treat your first prompt as a first draft, never as the final version.

Concept

Five Ingredients, One Instruction

01Role / Context

Who should the model act as, and what situation are we in?

02Task

The one specific action you want performed, stated as a verb.

03Input Data

The actual content to work on — text, data, or a question.

04Constraints & Format

Length, tone, structure, and what to leave out.

05Examples (optional)

A sample of the output you want, when words aren't enough.

06Success Criteria

How you — or the model — will know the answer is good enough.

Concept

Vague In, Vague Out

Weak draft Write something about our product.↖ which product? what output?
Revision Write a 3-sentence product description for a stainless-steel water bottle, for an e-commerce listing, in a friendly upbeat tone, under 40 words.
  • The weak prompt has no audience, no format, no length, and no tone.
  • The revision specifies medium, length, tone, and subject in one sentence.
  • Specificity isn't about writing more — it's about removing every open question the model would otherwise have to guess at.

Concept

Two Voices, Two Jobs

System.
"You are a patient technical writing tutor. Always explain concepts using a real-world analogy before the formal definition."
User.
"Explain what a variable is."
Assistant.
"Think of a variable like a labeled box you can put things in and swap out later... Formally, a variable is a named location that stores a value."
  • System prompt sets standing rules — persona, tone, boundaries — and persists across the whole conversation.
  • User prompt is the specific request for this turn — it changes every message.

Concept

Sequence Changes the Answer

  • Put the most important instruction first — and briefly repeat it at the end of long prompts.
  • Use delimiters — triple quotes, XML-style tags, or headers — to separate instructions from data.
  • Structure long prompts in order: role → task → data → constraints → question.
  • Numbered steps for multi-part tasks reduce the model skipping or merging steps.

Summarize the text below in 2 bullet points.

Watch For This

Common Pitfalls

✕

Too Vague"Write about X" leaves every decision to the model.

✕

One Prompt, Five JobsStacking unrelated tasks in one instruction confuses priority.

✕

No Format SpecifiedThe model guesses at length and structure — it won't fit your use case.

✕

Assuming MemoryThe model won't recall what you meant three messages ago unless it's in context.

✕

Contradictory Instructions"Be concise" and "cover everything in detail" fight each other.

✕

Burying the AskThe real question hidden in paragraph three of a long prompt gets missed.

Recap

What We Revised

  • A prompt is a specification: role, task, data, constraints, and — sometimes — examples.
  • Clarity beats cleverness — specificity removes guesswork.
  • System prompts set standing rules; user prompts ask the specific question.
  • Order matters: lead with the important instruction, use delimiters for structure.
  • Every good prompt is a revised draft, not a first attempt.
