Concept

What Is a Custom GPT / Assistant?

A custom GPT (ChatGPT) or Assistant (built via API) packages a system prompt, optional knowledge files, and optional tools into one reusable, nameable persona — instead of retyping setup instructions in every new chat.

Ordinary Chat

Re-explain every session

You paste context, tone rules, and examples each time you open a new chat

Custom GPT / Assistant

Configured once, reused forever

Instructions, files, and tools are attached permanently — anyone can open it and get the same behavior

Example: a "Support Triage" GPT that always replies in a set tone, checks a policy doc, and can look up an order status.

Concept

Instructions: The Heart of a Custom GPT

Instructions are the system prompt that defines what the persona does, how it behaves, and where it stops. Good instructions are specific enough to be reliable, not so long they bury the point.

Role

"You are a..."

State the job clearly: a study buddy, a code reviewer, a triage assistant

Scope & Boundaries

What it will and won't do

e.g. "Only answer questions about our product; decline anything else"

Tone

Voice & format

Formal or casual, bullet points or prose, reading level

Refusals & Escalation

What happens at the edge

e.g. "If unsure, say so and suggest contacting a human"

Concept

Knowledge Files: Grounding the Persona

Knowledge files let a custom GPT/Assistant reference your own documents instead of relying only on what it learned during training — reducing guessing on facts specific to you.

What Goes In

Docs, PDFs, style guides

Policy manuals, product docs, FAQs, brand style guides

Limits

Size & file-type limits apply

Platforms cap file count/size — keep files focused, not a dumping ground

Use Case

Consistent, sourced answers

e.g. a policy GPT that quotes the actual refund policy, not a guess

Caution

More files ≠ better

Irrelevant or outdated files add noise and can confuse retrieval

Concept

Actions & Tools: From Talking to Doing

Actions (ChatGPT, defined via an OpenAPI schema) and tool/function calling (Assistants API) let the persona call a real external system — a database, a calendar, an internal API — and use the result in its reply.

01

User asks a question

02

GPT decides to call an action

03

Action executes & returns data

04

GPT answers using the result

Example: "What's the status of order #4471?" triggers an action that queries the order system, and the GPT reports the real status back.

  # stand-in for a real order-lookup API

The model decides when to call get_order_status based on the user's question — exactly what an Action does inside a real custom GPT.

Common Pitfalls

Where Custom GPTs Go Wrong

  • Leaking instructions — clever users can often get a GPT to reveal its system prompt; don't put secrets there
  • Knowledge file bloat — dumping every document in "just in case" adds noise, not accuracy
  • Scope too broad — a GPT that tries to do everything ends up mediocre at all of it
  • Skipping edge-case testing — test refusals and off-topic questions, not just the happy path
  • Actions with no error handling — a failed API call should produce a clear message, not a silent wrong answer
