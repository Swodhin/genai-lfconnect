Concept 01

Why Memory Matters

  • LLM calls are stateless by default — each request starts fresh
  • Without memory, every turn forgets prior tool results and decisions
  • Short-term memory = the running conversation and observations within one task
  • Long-term memory = facts or summaries stored across sessions (files, databases, vector stores)
  • Today's build uses short-term, in-loop memory only

Concept 02

Two Kinds of Memory in an Agent

In This Task

Working Memory

The list of messages and tool results within one run of the loop. Lost when the run ends unless you save it.

Across Sessions

Persistent Memory

User preferences and past results saved between runs — a file, a database, a vector store.

Concept 03

Framework or Roll Your Own?

Plain Python

Custom Loop

  • Transparent — you see every message and decision
  • Few dependencies, easy to debug
  • Best for learning and small, focused tasks
e.g. LangChain

Framework

  • Built-in tool routing, memory, many integrations
  • More abstraction and dependencies
  • Best once you're scaling past a simple loop

Guidance: start custom to understand the loop — reach for a framework when you outgrow it.

Concept 04

Anatomy of a Minimal Agent

  • A system prompt describing the goal and available tools
  • A loop that sends messages, checks for a tool call or a final answer
  • If a tool call: execute it, append the result, and repeat
  • A stopping condition — a final answer, or a max-turns limit

Concept 05

Guardrails for a Real Loop

  • A max-turns limit, so the loop always ends
  • A timeout, in case any single step hangs
  • Validate tool arguments before running them
  • Log every step — plan, act, observe — for debugging
  • Human-in-the-loop for risky actions: spending money, sending messages

Pitfalls & Misconceptions

Where Builds Go Wrong

No Turn Cap

Without a limit, a bug becomes an infinite loop and a runaway bill.

Unhandled Errors

One failing tool call crashes the whole agent instead of returning an error message.

Dropped History

Forgetting to append a tool result breaks the model's memory of what happened.

Framework Too Soon

Reaching for a heavy framework before understanding the loop it wraps.

Recap

Key Takeaways

  • Memory is just the message list you carry through the loop
  • Start with a custom loop — reach for a framework once you outgrow it
  • A minimal agent = system prompt + loop + tools + a stopping condition
  • Guardrails (turn limits, validation, logging) are what make a loop production-ready
