Concept 01

A Single Prompt vs. an Agent

One Shot

Single Prompt

  • You ask once, the model answers once
  • No memory of tools it used or steps taken
  • You decide what happens next
  • Great for: writing, summarizing, one-off Q&A
Looping

Agent

  • Works toward a goal across multiple steps
  • Chooses its own actions, including tool calls
  • Observes results and decides what to do next
  • Great for: multi-step tasks with changing information

Concept 02

A Working Definition

DEFINITION. An agent is a system that uses an LLM to decide what actions to take, in a loop, in pursuit of a goal — with less step-by-step supervision than a single prompt.
  • Instead of: "Summarize this email" (one call, one answer)
  • An agent might: read the email → decide it needs more context → search the inbox → decide it has enough → draft a reply → send it
  • The model — not your code — is choosing the next step at each point

Concept 03

Autonomy Is a Spectrum

LVL 0

Fixed Script

No LLM decisions at all — a hardcoded sequence of steps.

LVL 1

Single Call

One prompt, one answer. You decide what happens next.

LVL 2

Approved Steps

Model picks a tool from a short list; a human approves each call.

LVL 3+

Full Loop

Model chooses tools and order; a human just checks the final result.

Concept 04

The Decision Loop (A Preview)

  • Plan — decide what to do next, given the goal and everything observed so far
  • Act — take that action: call a tool, write, search, respond
  • Observe — look at the result and decide: done, or loop again?
  • Today we zoom in on Plan only — the full mechanics of Act and Observe are next class

Concept 05

Useful and Risky

Upside

Why It's Useful

  • Handles multi-step tasks without a human in every loop
  • Adapts when new information shows up mid-task
  • Cuts down on manual "glue code" between steps
Downside

Why It's Risky

  • Can loop longer than expected — or forever, without a limit
  • Can take the wrong action confidently
  • Needs guardrails and human checkpoints for anything risky
  • Costs add up — every step is another model call

Recap

Key Takeaways

  • An agent uses an LLM to decide actions, in a loop, toward a goal
  • Autonomy is a spectrum — not a binary "agent or not"
  • Plan → Act → Observe is the heart of "agent-ness"
  • More autonomy means more power and more risk
  • Not every tool-using program is an agent
