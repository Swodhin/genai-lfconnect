Variables & Dynamic Typing

  • A variable is a name bound to a value — no type keyword required, just name = "Ada"
  • Python figures out the type at runtime; this is called dynamic typing
  • Reassigning a name can even change its type — legal, but a readability smell
  • Naming rules: letters, digits, underscores; can't start with a digit; case-sensitive
  • Convention: snake_case for variables — learning_rate, not LearningRate
model_name = "gpt-4" temperature = 0.7 is_ready = True temperature = "warming up" # legal, but confusing — avoid this

Making Decisions: if / elif / else

  • if runs its block only when the condition is True; indentation defines the block, not braces
  • elif checks another condition if the first was False; else catches everything else
  • Comparisons: == != < > <= >=
  • Combine conditions with and, or, not
  • Falsy values: 0, 0.0, "", [], {}, None, False — everything else is truthy
score = 82 if score >= 90: grade = "A" elif score >= 80: grade = "B" else: grade = "C" print(grade) # B

Repeating Work: for Loops

  • for walks through any iterable — a string, list, range(), dict, and more
  • range(5) produces 0,1,2,3,4 — start, stop, optional step
  • enumerate(items) hands you (index, value) pairs for free
  • The loop variable name is yours to choose — pick something readable
prompts = ["Hi", "Explain RAG", "Bye"] for i, prompt in enumerate(prompts): print(f"{i}: {prompt}")

Repeating Until Done: while Loops

  • while repeats as long as its condition stays True — good when you don't know the count in advance
  • break exits the loop immediately; continue skips to the next iteration
  • Forgetting to update the condition inside the loop causes an infinite loop
  • Common pattern: polling an API — while not response.done
attempts = 0 while attempts < 3: print(f"Attempt {attempts + 1}") attempts += 1 print("Done")

>>> concept 06

Ordered Collections: Lists vs Tuples

list — mutable

Square brackets. Grows, shrinks, and changes after creation: .append(), .remove(), .sort(). Use it when data changes.

history = ["hi"]
history.append("bye")

tuple — immutable

Parentheses. Can't change after creation — great for fixed groupings like coordinates.

point = (10, 20)
# point[0] = 99 # TypeError!
Both are ordered: indexing [0], slicing [1:3], and len() work on either.

>>> common pitfalls

Where Beginners Get Tripped Up

Indentation errors

Python uses whitespace, not braces. Mixing tabs and spaces breaks code.

== vs =

One compares, one assigns. if x = 5: is a syntax error on purpose.

Off-by-one in range()

range(5) stops before 5 — it never produces 5 itself.

Shared mutable references

Lists/dicts are mutable; copy with list(x) if you don't want to share one.
