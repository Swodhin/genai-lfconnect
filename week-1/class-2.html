Strings: Indexing, Slicing, Immutability

  • A string is an ordered sequence of characters, written with ' or " quotes
  • Index from 0: name[0] is the first character, name[-1] is the last
  • Slice with [start:stop]: name[0:3] grabs the first three characters
  • Strings are immutable — methods return new strings, they don't change the original
model = "gpt-4-turbo" print(model[0]) # 'g' print(model[-1]) # 'o' print(model[0:3]) # 'gpt' print(len(model)) # 11

Building Text with f-strings

  • Prefix a string with f and drop expressions inside { }
  • Any expression works inside the braces — even math or function calls
  • Format specifiers control decimals, commas, and percentages: {score:.1%}
  • f-strings are the standard way to build prompts dynamically
name = "Ada" tokens_used = 1532 print(f"{name} used {tokens_used:,} tokens today") # Ada used 1,532 tokens today

JSON: The Format APIs Speak

  • JSON (JavaScript Object Notation) is plain text shaped like nested dicts and lists
  • Every LLM API sends and receives JSON — request bodies, response bodies, errors
  • Type mapping: JSON object → Python dict, JSON array → Python list
  • JSON's true/false/null become Python's True/False/None automatically

Parsing JSON with json.loads

  • import json gives you the standard library's JSON toolkit
  • json.loads(text) turns a JSON string into a Python dict/list
  • Once parsed, it's just a normal dict — access with ["key"], loop with .items()
  • Nested data: chain the brackets — response["choices"][0]["message"]
import json raw = '{"role": "assistant", "content": "Hello!"}' data = json.loads(raw) print(data["content"]) # Hello!

Generating JSON with json.dumps

  • json.dumps(obj) turns a Python dict/list into a JSON string
  • indent=2 makes it human-readable for logs and debugging
  • sort_keys=True gives consistent, diffable output
  • Use this to build request bodies or save structured results to disk
import json payload = {"model": "gpt-4", "temperature": 0.7} print(json.dumps(payload, indent=2))

Reading & Writing Files

  • open(path, mode) opens a file — "r" read, "w" write, "a" append
  • Always use with open(...) as f: — it closes the file automatically, even on errors
  • f.read() grabs the whole file as one string
  • json.load(f) / json.dump(obj, f) read and write JSON straight from a file handle
with open("log.json", "w") as f: json.dump({"status": "ok"}, f, indent=2) with open("log.json") as f: data = json.load(f)

>>> common pitfalls

Where Beginners Get Tripped Up

Quotes inside JSON

JSON strings require double quotes. json.dumps() always outputs them, even from single-quoted Python strings.

true/false/null mix-ups

Writing Python's True into raw JSON text (instead of true) breaks parsers.

Forgetting immutability

s.replace(...) doesn't change s — it returns a new string you must reassign.

KeyError on missing fields

Use .get("key", default) instead of ["key"] when a field might be absent.
